# Weather
CLI-утилита и HTTP-сервер на Go для получения текущей погоды в любом городе с использованием API Яндекс.Погоды. Проект создан в учебных целях для изучения Go, работы с HTTP, JSON.

## Технологии
- [Go 1.22.2](https://go.dev/)
- [godotenv](https://github.com/joho/godotenv)
- [Echo](https://echo.labstack.com/)

## Требования
- Установить [Go](https://go.dev/dl/) версии 1.22+
- API-ключи для Яндекс.Погоды (бесплатный для "Умного дома") и Яндекс.Геокодер (бесплатный тариф)

## Установка
1. Клонируйте репозиторий:
```bash
git clone https://github.com/твой_аккаунт/Weather.git
cd Weather
```

2. Установить зависимости
```bash
go mod tidy
```

3. Создайте файл `.env` в корне проекта (используя `.env.example` как шаблон) и вставьте туда свои API-ключи.
(Их можно получить зарегистрировавшись в: `Яндекс погода для умного дома` и `Геокодер (API Яндекс Карты)`. Бесплатно.)

## Использование
CLI-версия:
```bash
go run cmd/cli/main.go Москва
```
Пример вывода:
```bash
Температура: 22
Ощущается как: 23
Состояние: cloudy
```
HTTP-сервер:
```bash
go run cmd/api/main.go
```
Сервер запустится на порту `8080`. Эндпоинт:
`GET /weather?city=НАЗВАНИЕ_ГОРОДА`
Параметр city необязателен. Если он не указан, сервер попытается определить ваш город по IP-адресу (с помощью внешнего сервиса ip-api.com). Если IP не публичный (локальный), будет возвращена ошибка.
Примеры ответа:
```json
{
  "status": "Success",
  "weather": {
    "city": "Москва",
    "temperature": 21,
    "feels_like": 21,
    "condition": "cloudy"
  }
}
```
```json
{
  "status": "error",
  "error": "город не найден"
}
```
